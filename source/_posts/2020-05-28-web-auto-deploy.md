## 传统的前端代码手工部署流程如下：

1. 打包 `npm run build` 生成 dist 目录

2. 链接服务器，切换路径到 web 目录

3. 代码上传到 web 目录

这个流程基本每次修改就要执行一遍，并且在忙起来的时候很容易出错，要么是生产环境上传了测试代码，要么是多个项目，文件目录出错等。

为了解决这个问题，也为了提升开发体验，所以打算写个脚本自动执行。

目标是，一行代码即可解决： 

```
npm run deploy
```
